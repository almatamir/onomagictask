# Claude Configuration for Guess The Country Project

## Code Style
- Write all code and comments in English only
- Keep code simple and readable 
- Make minimum changes when necessary
- Never rewrite a whole file if only one line needs to change
- Add comments only when the logic is not obvious
- Prefer clear variable names over short ones
- No unnecessary abstractions - keep files flat and simple
- Each file should do one thing only
- Don't refactor working code
- Don't add features that weren't asked for
- If something works, don't touch it

## Development Approach
- Work step by step, one file at a time
- Explain what each file does before writing it
- Explain why a solution works, not just what it does
- Ask before making big architectural changes
- When something breaks, explain the root cause before fixing it
- Never generate code for multiple files at once
- Check if the previous step worked before moving to the next

## Project Architecture
The project is split into two separate parts:

### Backend (Python + Flask)
- `app.py` - Flask entry point, registers blueprints
- `routes/game.py` - two endpoints: GET /api/country, POST /api/guess
- `services/database.py` - all MongoDB logic
- `services/gemini.py` - Gemini API via direct HTTP requests
- `seed.py` - one-time script to seed 30 countries into MongoDB

### Frontend (Next.js)
- `app/page.tsx` - single page, all UI logic in one file
- Communicates with backend via fetch to localhost:5000

## Project Stack
- Backend: Python 3.8 + Flask + PyMongo + certifi
- Frontend: Next.js + TypeScript + Tailwind CSS
- Database: MongoDB Atlas (free tier)
- AI: Gemini API via direct HTTP requests (not the SDK - incompatible with Python 3.8)
- Backend runs on port 5000, frontend on port 3000/3001

## Key Technical Decisions
- Gemini is called only when user clicks "Next Country" (?new=true)
- If Gemini fails, always fall back to MongoDB
- MongoDB is pre-seeded with 30 countries as a safety net
- New countries generated by Gemini are saved to MongoDB automatically
- SSL certificates fixed using certifi library (required on Mac)

## Project Rules
- Never commit .env files
- Always activate venv before running Python
- MongoDB fallback must always work even if Gemini is down
- Keep frontend and backend completely separate folders
- No Hebrew in code, comments, or variable names
